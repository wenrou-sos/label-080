# 已选 Feature：技师绩效排行榜

## 完整提示词

---

我现在这个足疗店管理系统前台页面上，想加一个"技师绩效"的功能。具体需求是这样的：

在 /reception 页面已有的控制台 tab 旁边（或者页面上方加个 tab 切换），新增一个"技师绩效"视图。这个页面要统计每个技师的今日工作表现，按贡献营收从高到低排成一个排行榜。

数据来源就用现有的 sessions、services 和 technicians。计算逻辑是：遍历今天所有的上钟记录，对每个技师汇总——上钟次数（完成了几个服务）、服务总时长（分钟）、贡献营收（把对应的服务价格加起来）、服务过的顾客人数。如果有还没结束的上钟，按当前时间算已进行的时长，但营收只算已结束的（endSession 时累计到 stats 里）。

排行榜用一个卡片列表展示，每张卡片包含：
- 排名序号（前三名显示 🥇🥈🥉 奖牌图标，用 lucide-react 的 Medal / Trophy）
- 技师头像（取名字第一个字，圆形容器，背景色按排名渐变）
- 技师姓名
- 四个关键指标：上单数、总时长、营收、顾客数
- 每个指标用一个小型横向柱状条显示在同级中的占比（最宽的 100% 是当前最高值）
- 排名变化指示：在卡片右上角显示 ↑ 或 ↓，旁边带变化位数（比如 ↑2 表示比上次查看时上升了2名）。第一次加载或没有历史排名时不显示。

排名变化需要记录上一次查看时的排名快照。可以用一个新的 store slice（比如 `lastRankSnapshot`），每次切换到绩效 tab 时，把当前排名存下来，下次切回来时对比差异。快照存到 localStorage 持久化，刷新不丢失。

排行榜顶部加一个统计摘要条：总技师数、今日总上单数、总服务时长、最高营收技师名字。

整个绩效视图用深色主题风格跟现有前台页面保持一致，卡片之间用 grid 布局，一行2-3张。

大概要改的文件：
- `src/types/index.ts`：加几个统计相关的类型
- `src/store/index.ts`：加 ranking 相关的状态和快照逻辑
- `src/pages/ReceptionPage.tsx`：加 tab 切换逻辑
- `src/pages/TechnicianRanking.tsx`：新建排行榜页面组件
- `src/components/TechnicianRankCard.tsx`：新建排行卡片组件
